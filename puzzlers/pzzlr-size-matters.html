Size Matters

Contributed by Andrew Phillips
Source Paul Phillips
Tested with Scala version 2.11.0-RC3

What is the result of executing the following code?

  import collection.mutable
  def howManyItems(lunchbox: mutable.Set[String]): Int = (lunchbox + "apple").size
  println(howManyItems(mutable.Set("chocolate bar", "orange juice", "sandwich")))

  def howManyItemsRefac(lunchbox: mutable.Iterable[String]): Int = (lunchbox + "apple").size
  println(howManyItemsRefac(mutable.Set("chocolate bar", "orange juice", "sandwich")))
  1. Prints: 
    4
4
  2. Prints: 
    3
4
  3. Prints: 
    4
3
  4. Prints: 
    4
47

Explanation

Whilst the + operator is defined for Sets, with the effect of adding the provided argument to the set, it is not defined for collections in general.

+ is also Scala's string concatenation operator, however! And since Scala by default provides an implicit conversion from Any to String, the compiler can make lunchbox + "apple" compile also in the case of howManyItemsRefac. Rather than adding the apple to the lunchbox, however, the statement ends up computing the concatenation lunchbox.toString + "apple" and returning the length of the resulting string.

The implicit view that enables the Any to String conversion, Predef.any2stringadd, is provided for interoperability with Java. Due to its ability to cause unexpected effects such as this one, it continues to provoke lively debate.

In compiled classes, as well is in the REPL when using the :paste -raw mode, you can disable this (or any other implicit, for that matter), by starting adding the following import to the beginning of your program: 

import Predef.{any2stringadd => _}
